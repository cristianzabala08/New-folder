Files & Folder Structure

All files & folders are well organized of Bible application. You can see below images to get some idea about files & folders. Bible is developed on the top of Codeigniter HMVC pattern. So all major files are located in modules folder inside application folder. Inside modules folder there are three folders access, dashboard and home. Which are representing as regiser/login/social login, dashboard files and all home/front website files. These three folders has individual controllers folder and views folder.

Get Started

First download your fresh updated copy of Bible from Codecanyon. You will get a zip file called main.zip extract this file. After extract you will see files and folders like below image.

Now go to your cpanel area and select file manager. Inside file manager select public_html from left side menu. Now click upload button from top bar and upload all the files and folders from your Bible application.

Now go to your cpanel area again and select MySQL Database. Inside MySQL database create a new database and a new user. Combine new database and new user together. Remember your database name, user name and password for later. Go to your Phpmyadmin panel and select your database which you created recently. Click on import button to import database from application. Browse inside Bible folder and import database-final.sql and import. All data will be imported and you are almost done.

Now go to application folder to config database and base url. Path application -> config -> config.php and database.php. Edit config.php and insert your base url in line 27. If you are in localhost then put your localhost site path ex: $config['base_url'] = 'http://localhost/yoursite/'; or if you are in live site then put ex: $config['base_url'] = 'http://yourdomain.com/'; Base url is very important please don't ignore this. Without proper base url your site assets will not load properly.

You are done and ready to use your church application. Browse http://yoursite.com/ to see your site.

You can access your dashboard by browsing http://yoursite.com/access/login or http://yoursite.com/login or http://yoursite.com/dashboard. Admin login info is Email : admin@site.com and Password : 123456 and User login info is Email : user@site.com and Password : 123456

Dashboard -> Website

Website module is for frontend/website/home section. Which you can see at the first when you browse your site url. In this module there are three six sub-module.

  • Basic Info - is for all basic info including site title, slogan, logo, favicon, address, google map, site currency, admin notification email, theme color etc.

    Recomended logo and favicon size is 50kb, larger than this size could affect your site loading speed. Also notice your logo width and height, try to keep logo width uner 100px larger than this could break css of your site.
  • Menus - is for site menus. Where you can add new menus, edit menus, delete menus and also you can drag and drop menus to change the position of menus. Drag and drop will not work if you want to change sub menu to parent menu or parent menu to sub menu.
  • Pages - is for creating pages and link them with menu links. You can create unlimited pages. All pages works like Wordpress page.
  • Slider - is for adding slider to website header. You can change position of slider by drag and drop. Slider support multiple images up to 20 by default.
  • Gallery - is for website gallery images. Add or delete gallery images from here. Gallery also support multiple image upload at once upto 20 by default.
  • Section - is the coolest thing there. You can add multiple section in your site. Every section support background image, text, link and shortcode.
  • Sidebar Hide/Show - to hide/show the sidebar please click on the arrow icon beside the right side of logo. One click to hide the sidebar and double click to show the sidebar.
  • File/Image Upload - to upload profile photo or any other photo keep eye on your image/file size. Specially on images, if your image is too large than 1-2 MB then your site loading speed could affect. Also you maybe find data saving problem. If you need to uplaod large image than increase your hosting memory limit to 256 MB otherwise image will not show in cropping editor area, in image cropping area all images converted to base64 format string that's why very large image need bigger memory to store and to crop perfectly.

Dashboard -> Social Login Setup

Social login is a very important feature in this social media era. That's why Bible support social media login and signup. Social media sign-in and sign-up is a little bit tricky. To get this feature you need to setup your social media api keys. We are using HybridAuth to bring this feature in Bible. So if you want to know more about that you can see its official documents. https://hybridauth.github.io/

First go to application -> config -> hybridauth.php. In this file you can see there are nine providers or social media which you can use for your application. But currently Bible supports three social media or providers eg: Facebook, Twitter and Google. Each of them need their own api key and secret. You need to get those api key and secret from your social media.

Facebook Setting

Go to Facebook Developer Page https://developers.facebook.com to get api key and secret key. Also config your application with valid redirect urls. See example image below. If your redirect url is not valid then social login would not work.

Copy http://yoursite.com/access/login/endpoint?hauth_done=Facebook url and insert into your facebook apps config page field as shown below and replace with your site url. If your application in a sub folder then don't forget to add in the url after http://yoursite.com/your-subfolder-name/

Twitter Setting

Go to Twitter Developer Page https://apps.twitter.com/ to get api key and secret key. Also config your application with valid redirect urls. See example image below. If your redirect url is not valid then social login would not work.

Copy http://yoursite.com/access/login/endpoint?hauth_done=Twitter url and insert into your twitter apps config page field as shown below and replace with your site url. If your application in a sub folder then don't forget to add in the url after http://yoursite.com/your-subfolder-name/

Google Setting

Go to Google Console https://console.developers.google.com/apis/ to get api key and secret key. Also config your application with valid redirect urls. See example image below. If your redirect url is not valid then social login would not work.

Copy http://yoursite.com/access/login/endpoint?hauth_done=Google url and insert into your google apps config page field as shown below and replace with your site url. If your application in a sub folder then don't forget to add in the url after http://yoursite.com/your-subfolder-name/
